EmailJS Setup Guide for Rapido Frontend

To enable the Contact Form to send emails to support@genzoride.com, follow these steps:

1. Create an EmailJS Account

  1. Go to https://www.emailjs.com/ and create a free account.

2. Connect Email Service

  1. In the EmailJS dashboard, go to the "Email Services" tab.
  2. Click "Add New Service".
  3. Select your email provider (e.g., Gmail, Outlook).
  4. Connect your account and click "Create Service".
  5. Ideally, use the `support@genzoride.com` email account if possible, or any email that you want to send *from*.
  6. Copy the Service ID (e.g., `service_xc8s9d`) and paste it into your `.env` file as `VITE_EMAILJS_SERVICE_ID`.

3. Create Email Template

  1. Go to the "Email Templates" tab.
  2. Click "Create New Template".
  3. Set the subject to something like: `New Contact Form Submission from {{name}}`.
  4. In the content, use the following variables ensuring they match the `name` attributes in your form input fields:
    • {{name}} - The sender's name
    • {{email}} - The sender's email
    • {{mobile}} - The sender's phone number
    • {{role}} - User role (Customer/Pilot)
    • {{message}} - The comment/message
  5. In the "To Email" field on the right, enter: support@genzoride.com.
  6. The "From Email" is typically your connected service email.
  7. Save the template.
  8. Copy the Template ID (e.g., `template_8szd7x`) and paste it into your `.env` file as `VITE_EMAILJS_TEMPLATE_ID`.

4. Get Public Key

  1. Go to the "Account" (or your name in the corner) -> "API Keys" (or just "Account").
  2. Copy the Public Key (e.g., `user_8s7d6f87s6d`).
  3. Paste it into your `.env` file as `VITE_EMAILJS_PUBLIC_KEY`.

5. Update .env File

Open the `.env` file in your project root and replace the placeholders with your actual IDs:

VITE_EMAILJS_SERVICE_ID=service_xxxxx
VITE_EMAILJS_TEMPLATE_ID=template_xxxxx
VITE_EMAILJS_PUBLIC_KEY=public_xxxxx

6. Restart Server

After updating `.env`, stop the terminal (Ctrl+C) and run `npm run dev` again to load the new variables.
